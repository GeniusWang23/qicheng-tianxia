# 《器成天下走》开发者指南

> 本文档写给想自己改代码的人。即使你完全没写过代码，也能从这里入手。

---

## 一、项目框架总览

### 文件清单

```
器成天下走游戏/
├── index.html            ← 游戏本体（所有界面 + 样式 + 逻辑都在这一个文件里）
├── launcher.py           ← 启动器（用浏览器打开游戏）
├── 器成天下走.exe         ← 打包好的可执行文件（双击即玩）
├── README.txt            ← 使用说明
├── 游戏介绍与设计理念.md   ← 设计文档
├── 开发者指南.md          ← 本文档
├── index.html.bak        ← 原始代码备份（如果你改坏了可以恢复）
└── launcher.py.bak       ← 启动器备份
```

**最重要的文件只有一个：`index.html`。** 游戏的全部内容——画面、样式、玩法逻辑——都在这个文件里。你只需要改这一个文件。

---

### index.html 内部结构（从上到下）

```
┌─────────────────────────────────────┐
│ ① <head> + <style>                  │  ← 外观样式（颜色、字体、布局）
│    古风配色、长卷地图、响应式布局     │
├─────────────────────────────────────┤
│ ② HTML 骨架                         │  ← 页面结构（按钮、文字、面板）
│    导航、大地图、库房面板、订单面板    │
├─────────────────────────────────────┤
│ ③ <script> —— JS 游戏脚本           │  ← 玩法逻辑（这就是核心）
│                                       │
│  ├── CONFIG         常量配置          │  改数值：采集量、价格、消耗等
│  ├── SEASONS        四季数据          │  改季节效果或新增季节
│  ├── GameState 类   游戏状态管理      │  存档、属性计算
│  ├── DOM 工具       页面元素操作      │  显示提示、写日志
│  ├── render()       画面刷新          │  把所有数据显示到屏幕上
│  ├── 游戏操作函数    按钮触发的功能    │  采集/制坯/烧窑/交付/招募/升级
│  └── 初始化代码      游戏启动时的逻辑  │  读取存档或开始新游戏
└─────────────────────────────────────┘
```

## 二、核心概念（零基础必读）

### 游戏状态 = 一本账本

游戏里所有数据都记在 **`G`** 这本账上。打开浏览器的开发者工具（F12），在控制台输入 `G` 回车，你就能看到当前所有的游戏数据：

```
G → {
  silver: 80,     // 银两
  fame: 0,        // 声望
  quality: 0,     // 品质（累计）
  clay: 12,       // 高岭土数量
  wood: 10,       // 松柴数量
  glaze: 5,       // 釉料数量
  blank: 0,       // 素坯数量
  ware: 0,        // 成瓷数量
  workers: 2,     // 匠人数
  ...
}
```

**游戏的过程就是：** 你点按钮 → 账本上的数字变化 → 屏幕刷新显示新数字。

### 三个关键概念的关系

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│ CONFIG   │  →   │    G     │  →   │ 页面显示  │
│ (配方书)  │      │ (账本)   │      │ (屏幕)    │
└──────────┘      └──────────┘      └──────────┘
   改这里           运行中变化         自动刷新
   （静态）         （动态）
```

- **CONFIG** = 配方书。写着"做坯需要 3 瓷土 + 1 釉料"、"招募匠人需要 35 银两"——这些是固定的数值。
- **G** = 账本。记录你当前有多少钱、多少资源。游戏过程中不断变化。
- **页面显示** = 屏幕。账本变了，屏幕自动更新。

## 三、如何修改数值（最简单的改动）

找到 `CONFIG` 那一块，直接改数字就行。

> 用记事本打开 `index.html`，找到第 40 行附近：

```javascript
const CONFIG = {
  GATHER:    { clay:6, wood:5, glaze:3 },         // ← 每次采集获得的数量
  PRODUCE:   { clayCost:3, glazeCost:1, ... },    // ← 制坯消耗的原料
  FIRE:      { woodBase:2, ... },                  // ← 烧窑消耗的柴火
  ORDER:     { baseSilver:45, ... },               // ← 订单报酬
  IMPERIAL:  { wares:4, qualityMin:16, ... },      // ← 御命要求
  HIRE:      { cost:35 },                          // ← 招募费用
  UPGRADE:   { cost:90 },                          // ← 升级费用
```

**例子：想让招募匠人更便宜**
```
把  HIRE: { cost:35 }
改成  HIRE: { cost:20 }
```
保存文件，浏览器刷新一下，招募就只要 20 银两了。

**例子：想让秋季品质加成更高**
```
把 autumnBonus:2
改成 autumnBonus:5
```
秋季烧窑的品质加成就从 +2 变成 +5。

## 四、如何修改季节效果

找到第 52~57 行：

```javascript
const SEASONS = [
  { name:"春", desc:"风和日暖，晒坯顺遂", woodExtra:0, qualityBonus:0, produceSlow:false },
  { name:"夏", desc:"梅雨渐起，坯体晾晒变慢", woodExtra:0, qualityBonus:0, produceSlow:true },
  { name:"秋", desc:"柴火干爽，窑温更稳", woodExtra:0, qualityBonus:2, produceSlow:false },
  { name:"冬", desc:"寒风入窑，柴火消耗增加", woodExtra:1, qualityBonus:0, produceSlow:false }
];
```

每个季节有 4 个属性：

| 属性 | 含义 | 可选值 |
|------|------|--------|
| `name` | 季节名称 | 任意文字，比如 "梅雨季" |
| `desc` | 描述文字，显示在游戏顶部 | 任意文字 |
| `woodExtra` | 烧窑额外消耗的柴火数 | 0 或 1 或更多 |
| `qualityBonus` | 烧窑的品质加成 | 0 或 2 或更多 |
| `produceSlow` | 是否制坯减速（夏季梅雨） | `true` 或 `false` |

**例子：新增一个"梅雨季"替换夏季**
```javascript
{ name:"梅雨季", desc:"连日阴雨，柴火受潮", woodExtra:1, qualityBonus:0, produceSlow:true }
```
这样梅雨季烧窑要多费柴火，制坯也慢。

## 五、如何新增功能（以"新增一个工匠技能"为例）

假设你想加一个新功能：匠人达到 5 人时，烧窑品质额外 +1。

### 第 1 步：在 CONFIG 里加一个开关

找到 `CONFIG`，加一行：

```javascript
const CONFIG = {
  // ... 原有的 ...
  WORKER_BONUS: { count:5, qualityBonus:1 },   // ← 新增：匠人达到5人时品质+1
```

### 第 2 步：在 fireKiln 函数里加判断

找到 `function fireKiln()`（第 146 行），在计算品质的地方做修改：

```javascript
// 改之前：
const q = CONFIG.FIRE.qualityBase + G.kiln + G.seasonInfo.qualityBonus;

// 改之后：
let q = CONFIG.FIRE.qualityBase + G.kiln + G.seasonInfo.qualityBonus;
if (G.workers >= CONFIG.WORKER_BONUS.count) {
  q += CONFIG.WORKER_BONUS.qualityBonus;     // 匠人够多，品质再加
}
```

### 第 3 步：保存并测试

保存 `index.html`，刷新浏览器，招募到 5 个匠人后烧窑，品质应该多 +1。

### 原则总结

```
新增功能 = 
  ① 在 CONFIG 里定义数值（方便以后改）
  ② 在 GameState 里添加属性（如果需要记录新数据）
  ③ 在某个操作函数里写逻辑（按钮触发什么）
  ④ 在 render() 里显示新数据（如果需要显示在屏幕上）
```

## 七、launcher.py 是什么

`launcher.py` 是一个极简的 Python 程序，只在你的电脑上开一个临时的网页服务器，然后用浏览器打开游戏。

它跟游戏逻辑无关。如果你只是改游戏数值，**不需要动这个文件**。

如果你想改启动方式：
- 改端口号：把 `0` 改成 `8080` 就会用固定端口 `http://127.0.0.1:8080/`
- 改浏览器：在 `webbrowser.open()` 那行换成你想要的浏览器路径

## 八、如何测试你的修改

1. **改数值**：用记事本打开 `index.html`，找到对应位置修改，保存。
2. **刷新**：如果游戏已经在浏览器中打开，按 F5 刷新即可。
3. **如果改坏了**：把 `index.html.bak` 复制一份改名为 `index.html` 就能恢复。
4. **调试小技巧**：在浏览器按 F12 打开"开发者工具" → 点击"控制台"(Console) → 输入 `G` 回车，就能看到当前所有数据。

## 九、如果你想做更复杂的改动

以下改动需要理解 JavaScript 基础（变量、函数、if 判断），但框架已经为你准备好了：

### 新增一种资源（如"颜料"）

需要在 **4 个地方** 添加：

```
① CONFIG.INIT 里加一笔：pigment:0
② HTML 里库房面板加一行显示：<span>颜料 <b id="pigment">0</b></span>
③ 新增一个采集按钮：<button onclick="gather('pigment')">采颜料</button>
④ CONFIG.GATHER 里加：pigment:3
⑤ 在制坯或绘制环节消耗颜料
```

### 新增一种操作（如"画坯"）

```
① CONFIG 里定义消耗和产出
② 写一个新函数 function paint() { ... }
③ HTML 里加一个按钮 <button onclick="paint()">画坯</button>
④ 在 render() 里更新相关显示
```

### 新增御命之外的订单类型

```
① CONFIG 里新增一个配置块
② 在 deliver() 里加判断分支
③ 在 render() 里显示新订单信息
```

## 十、快速参考：每个函数是干什么的

| 函数名 | 触发方式 | 作用 |
|--------|----------|------|
| `gather(type)` | 点击采集按钮 | 增加对应资源 |
| `produce()` | 点击"制作坯体" | 消耗瓷土+釉料，产出素坯 |
| `fireKiln()` | 点击"入窑烧制" | 消耗素坯+柴火，产出成瓷+品质 |
| `deliver()` | 点击"交付订单" | 消耗成瓷，获得银两+声望 |
| `hire()` | 点击"招募匠人" | 消耗银两，增加匠人 |
| `upgrade()` | 点击"升级镇窑" | 消耗银两，提升窑炉等级 |
| `nextDay()` | 点击"度过一日" | 推进时间，检查御命超时 |
| `resetGame()` | 点击"重开长卷" | 重置所有数据 |
| `render()` | 自动调用 | 刷新屏幕显示 |
| `showToast()` | 自动调用 | 显示底部弹出提示 |
| `addLog()` | 自动调用 | 写入掌柜手记 |
